src.rag_toolkit.core.vectorstore¶

Core vector store abstraction.

This module defines the Protocol interface that all vector store implementations must satisfy, enabling seamless switching between different vector databases (Milvus, Pinecone, Qdrant, Weaviate, etc.) without changing application code.

Design Philosophy:

  • Protocol-based: No inheritance required, duck typing with type safety

  • Provider-agnostic: Works with any vector database

  • Minimal interface: Only essential operations

  • Metadata support: Rich filtering and search capabilities

Bases: Protocol

Protocol for vector store operations.

This defines the interface that all vector store implementations must provide. It abstracts common operations across different vector databases.

Implementations:

  • MilvusVectorStore: Milvus 2.x implementation

  • PineconeVectorStore: Pinecone cloud implementation (future)

  • QdrantVectorStore: Qdrant implementation (future)

Example

>>> store: VectorStoreClient = MilvusVectorStore(host="localhost")
>>> store.create_collection("docs", dimension=384)
>>> ids = store.insert(
...     collection_name="docs",
...     vectors=[[0.1, 0.2, ...], ...],
...     texts=["doc 1", "doc 2"],
...     metadata=[{"source": "a"}, {"source": "b"}]
... )
>>> results = store.search(
...     collection_name="docs",
...     query_vector=[0.1, 0.2, ...],
...     top_k=5
... )

Create a new collection/index for storing vectors.

Parameters:

  • dimension (int) – Vector dimension (e.g., 384, 768, 1536)

  • metric (str) – Distance metric (“IP”=inner product, “L2”=euclidean, “COSINE”)

  • description (str | None) – Optional collection description

  • **kwargs (Any) – Provider-specific parameters

Raises:

  • CollectionExistsError – If collection already exists

  • ValueError – If dimension <= 0 or invalid metric

Example

>>> store.create_collection(
...     name="my_docs",
...     dimension=384,
...     metric="IP",
...     description="Product documentation embeddings"
... )

insert(collection_name, vectors, texts, metadata, *, ids=None, **kwargs)[source]¶

Insert vectors with associated text and metadata.

Parameters:

  • collection_name (str) – Target collection

  • vectors (list[list[float]]) – List of embedding vectors (must match collection dimension)

  • texts (list[str]) – Source texts for each vector

  • metadata (list[rag_toolkit.core.types.VectorMetadata]) – Metadata dict for each vector (for filtering)

  • ids (list[str] | None) – Optional custom IDs (auto-generated if None)

  • **kwargs (Any) – Provider-specific parameters

Returns:

List of assigned IDs (in same order as input)

Raises:

  • CollectionNotFoundError – If collection doesn’t exist

  • ValueError – If vectors/texts/metadata lengths don’t match

  • DimensionMismatchError – If vector dimension doesn’t match collection

Return type:

list[str]

Example

>>> ids = store.insert(
...     collection_name="docs",
...     vectors=[[0.1] * 384, [0.2] * 384],
...     texts=["First doc", "Second doc"],
...     metadata=[
...         {"source": "manual", "page": 1},
...         {"source": "api", "page": 5}
...     ]
... )
>>> print(ids)  # ["id_001", "id_002"]
search(collection_name, query_vector, top_k=10, *, filters=None, **kwargs)[source]¶

Search for similar vectors using ANN (approximate nearest neighbors).

Parameters:

  • collection_name (str) – Collection to search

  • query_vector (list[float]) – Query embedding vector

  • top_k (int) – Number of results to return

  • filters (rag_toolkit.core.types.VectorMetadata | None) – Metadata filters (e.g., {“source”: “manual”})

  • **kwargs (Any) – Provider-specific parameters (e.g., nprobe, ef)

Returns:

List of search results ordered by similarity (best first)

Raises:

  • CollectionNotFoundError – If collection doesn’t exist

  • DimensionMismatchError – If query dimension doesn’t match collection

Return type:

list[rag_toolkit.core.types.SearchResult]

Example

>>> results = store.search(
...     collection_name="docs",
...     query_vector=[0.15] * 384,
...     top_k=5,
...     filters={"source": "manual"}
... )
>>> for result in results:
...     print(f"ID: {result.id}, Score: {result.score:.4f}")
...     print(f"Text: {result.text[:100]}...")
hybrid_search(collection_name, query_vector, query_text, top_k=10, *, alpha=0.5, filters=None, **kwargs)[source]¶

Hybrid search combining vector similarity and keyword matching.

Parameters:

  • collection_name (str) – Collection to search

  • query_vector (list[float]) – Query embedding vector

  • query_text (str) – Query text for keyword search

  • top_k (int) – Number of results to return

  • alpha (float) – Weight for vector vs keyword (0=keyword only, 1=vector only)

  • filters (rag_toolkit.core.types.VectorMetadata | None) – Metadata filters

  • **kwargs (Any) – Provider-specific parameters

Returns:

List of search results with combined scores

Return type:

list[rag_toolkit.core.types.SearchResult]

Note

Not all vector stores support hybrid search natively. Default implementation may combine separate vector + keyword searches.

Example

>>> results = store.hybrid_search(
...     collection_name="docs",
...     query_vector=[0.15] * 384,
...     query_text="installation guide",
...     top_k=5,
...     alpha=0.7  # Favor vector search
... )
