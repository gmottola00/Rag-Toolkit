    Master these fundamental concepts to unlock the full potential of RAG Toolkit.

    RAG (Retrieval-Augmented Generation)¬∂

    What is RAG?

    RAG combines information retrieval with text generation to create more accurate, factual, and contextual responses.

    Traditional LLM¬∂

    graph LR
    A[üë§ User Query] --> B[ü§ñ LLM] --> C[üìù Response]
    style A fill:#e3f2fd
    style C fill:#ffebee

    Limitations

    • Limited to training data cutoff
    • Can hallucinate facts
    • No access to private/recent data
    • No source citations

    :material-brain-circuit: RAG-Enhanced LLM¬∂

    graph LR
    A[üë§ User Query] --> B[üîç Retrieval]
    B --> C[üíæ Vector Store]
    C --> D[üìö Relevant Docs]
    D --> E[üìã Context + Query]
    E --> F[ü§ñ LLM]
    F --> G[‚ú® Response]
    style A fill:#e3f2fd
    style G fill:#c8e6c9

    Benefits

    • Access to current information
    • Reduced hallucinations
    • Citations and sources
    • Private data integration
    • Always up-to-date knowledge

    The Building Blocks of RAG

    1. Embeddings¬∂

    Semantic Vector Representations

    Embeddings convert text into numerical vectors that capture semantic meaning.

    embedding_example.py
    1
2
3
    text = "Hello world"
vector = embedding.embed(text)
# [0.23, -0.45, 0.12, ...]  # 768 dimensions

    Semantic Similarity:

    Python
    1
2
    embed("dog") ‚âà embed("puppy")       # High similarity ‚úì
embed("dog") ‚â† embed("computer")    # Low similarity ‚úó

    Key Property

    Similar texts produce similar vectors ‚Äî enabling semantic search beyond keyword matching!

    • Dense Vectors

      Typically 384-4096 dimensions per text

    • Semantic Understanding

      Captures context, synonyms, relationships

    • Efficient Search

      Fast vector similarity operations

    • :material-language: Language Agnostic

      Works across multiple languages

    4. Chunking¬∂

    Document Segmentation

    Splitting documents into manageable pieces for optimal embedding and retrieval.

    chunking_example.py
    1
2
3
4
5
6
7

    document = "Very long document with lots of content..."

chunks = chunker.chunk(
    document,
    chunk_size=512,
    chunk_overlap=50,
)
# ["chunk1...", "chunk2...", "chunk3..."]

    Chunking Strategies:

    • Fixed Size

      Python
      1
2
3
      chunks = fixed_chunker.chunk(
    text, chunk_size=512
)

      Simple and consistent chunk sizes

    • Sentence-Based

      Python
      1
2
3
      chunks = sentence_chunker.chunk(
    text, max_sentences=10
)

      Respects semantic boundaries

    • Recursive

      Python
      1
2
3
      chunks = recursive_chunker.chunk(
    text, separators=["\n\n", "\n", ". "]
)

      Hierarchical splitting strategy

    • Token-Aware

      Python
      1
2
3
      chunks = token_chunker.chunk(
    text, max_tokens=800
)

      Respects LLM token limits

    Chunking Best Practices

    • Chunk size: 300-800 tokens optimal for most use cases
    • Overlap: 10-20% overlap preserves context at boundaries
    • Strategy: Match chunking to document structure (headings, paragraphs)

    ## Semantic Search vs Keyword Search

### Keyword Search

```python
query = "dog care"
results = keyword_search(query)
# Matches: "dog", "care", "dogs"

    Limitations: - Exact word matching - Misses synonyms - No context understanding

    Semantic Search (Embeddings)¬∂

    Python
    1
2
3
4
    query = "dog care"
query_vector = embed(query)
results = vector_search(query_vector)
# Matches: "pet grooming", "canine health", "puppy training"

    Advantages: - ‚úÖ Understands meaning - ‚úÖ Finds related concepts - ‚úÖ Language-agnostic - ‚úÖ Context-aware

class BaseEmbedding(ABC):
    @abstractmethod
    def embed(self, text: str) -> list[float]:
        pass

class MyEmbedding(BaseEmbedding):  # Must inherit!
    def embed(self, text: str) -> list[float]:
        return [0.0] * 768

    from typing import Protocol

class EmbeddingClient(Protocol):
    def embed(self, text: str) -> list[float]: ...

class MyEmbedding:  # No inheritance needed!
    def embed(self, text: str) -> list[float]:
        return [0.0] * 768

# Works seamlessly
pipeline = RagPipeline(embedding_client=MyEmbedding())

    Benefits: - ‚úÖ No inheritance required - ‚úÖ Duck typing with type safety - ‚úÖ Easy testing with mocks - ‚úÖ More flexibility

        self,
        embedding_client: EmbeddingClient,
        llm_client: LLMClient,
        vector_store: VectorStoreClient,
    ):
        self.embedding = embedding_client
        self.llm = llm_client
        self.store = vector_store

    Benefits: - ‚úÖ Easy to test (inject mocks) - ‚úÖ Flexible (swap implementations) - ‚úÖ Clear dependencies - ‚úÖ No global state

    # Check token count
def count_tokens(text: str) -> int:
    # Approximate: 1 token ‚âà 4 characters
    return len(text) // 4

# Fit within context window
max_context_tokens = 4096
prompt_tokens = 100
answer_tokens = 512

available_tokens = max_context_tokens - prompt_tokens - answer_tokens

# Select chunks that fit
selected_chunks = []
total_tokens = 0

for chunk in ranked_chunks:
    chunk_tokens = count_tokens(chunk.text)
    if total_tokens + chunk_tokens <= available_tokens:
        selected_chunks.append(chunk)
        total_tokens += chunk_tokens
